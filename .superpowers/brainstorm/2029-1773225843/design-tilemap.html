Feature 3: Tilemap Layers — Detailed Design

Tileset management, tile grid editing, two-mode workflow

Data Model

class Tileset:
    name: str                          # e.g. "Dungeon Tiles"
    tile_width: int                    # e.g. 16
    tile_height: int                   # e.g. 16
    tiles: list[np.ndarray]            # list of (tile_h, tile_w, 4) uint8 arrays
                                       # index 0 = empty/transparent tile (always)

    # Operations:
    add_tile(pixels) -> int            # returns new tile index
    remove_tile(index)                 # removes + reindexes references
    update_tile(index, pixels)         # edit tile content (live update)
    find_matching(pixels) -> int|None  # find tile with identical pixels
    import_from_image(img, tw, th)     # auto-slice image into tiles

class TileRef:
    index: int           # tile index in tileset (0 = empty)
    flip_x: bool         # horizontal flip
    flip_y: bool         # vertical flip
    # Packed as uint32: [flip_y:1][flip_x:1][unused:14][index:16]

class TilemapLayer(Layer):
    tileset: Tileset                   # reference to tileset
    grid: list[list[TileRef]]          # 2D grid of tile references
    grid_cols: int                     # canvas_width // tile_width
    grid_rows: int                     # canvas_height // tile_height
    edit_mode: str                     # "pixels" | "tiles"
    pixel_sub_mode: str                # "auto" | "manual"

    # Rendering: resolves grid → pixel buffer
    def render_to_pixels(self) -> np.ndarray:
        result = np.zeros((H, W, 4), dtype=np.uint8)
        for row, col in grid:
            ref = grid[row][col]
            if ref.index == 0: continue
            tile_pixels = tileset.tiles[ref.index]
            if ref.flip_x: tile_pixels = np.flip(tile_pixels, axis=1)
            if ref.flip_y: tile_pixels = np.flip(tile_pixels, axis=0)
            blit(result, tile_pixels, col * tile_w, row * tile_h)
        return result

    # Overrides Layer.pixels property for flatten_layers() compatibility
    @property
    def pixels(self) -> PixelGrid:
        return PixelGrid.from_array(self.render_to_pixels())

Two Editing Modes

Draw Pixels Mode (default)

Edit the tile content directly. All instances of that tile update live.

Auto Sub-Mode (default)

  • Draw pixels normally — feels like a regular layer
  • When you modify pixels in a tile cell, check if result matches existing tile
  • If match found → reuse that tile index
  • If no match → create new tile in tileset
  • Unused tiles auto-removed on save/cleanup

Manual Sub-Mode

  • Draw pixels → updates the tile definition directly
  • ALL instances of that tile update (Pyxel Edit live update)
  • No new tiles created, no reindexing
  • Best for refining a fixed tileset

Draw Tiles Mode

Place tiles from the tileset onto the grid. Doesn't modify tile content.

Tool Behavior

  • Pen: place selected tile at grid cell
  • Eraser: set cell to empty tile (index 0)
  • Fill: flood-fill region with selected tile
  • Pick: pick tile from canvas (set as active tile)
  • Rect: fill rectangle with selected tile
  • Line: place tiles in a line

Tile Transforms

  • Shift+X: flip selected tile horizontally
  • Shift+Y: flip selected tile vertically
  • Transforms stored per-cell (TileRef.flip_x/y)
  • Shown in tiles panel as toggle buttons

Mode Toggle: Shift+` (same as Pixelorama) or button in toolbar/context bar. Current mode shown in status bar: "PIXELS (Auto)" / "PIXELS (Manual)" / "TILES"

Canvas Overlay

When Tilemap Layer Active

  • Tile grid overlay drawn on canvas (dashed lines, subtle color)
  • Only visible at sufficient zoom (tile ≥ 8px on screen)
  • Current cell highlighted on mouse hover
  • In Tiles mode: cursor shows preview of selected tile

Grid Rendering

  • Drawn as part of canvas overlay (like pixel grid)
  • Cached — only redrawn on zoom/scroll change
  • Color: semi-transparent cyan to match theme
  • Can be toggled off in View menu

Tileset Import

def import_tileset_from_image(image_path, tile_w, tile_h) -> Tileset:
    """Import a tileset image and auto-slice into tiles."""
    img = Image.open(image_path).convert("RGBA")
    tileset = Tileset(name=..., tile_width=tile_w, tile_height=tile_h)
    tileset.tiles.append(np.zeros((tile_h, tile_w, 4)))  # index 0 = empty

    for y in range(0, img.height, tile_h):
        for x in range(0, img.width, tile_w):
            tile = np.array(img.crop((x, y, x+tile_w, y+tile_h)))
            # Skip fully transparent tiles
            if tile[:,:,3].sum() == 0: continue
            # Dedup: check if identical tile already exists
            existing = tileset.find_matching(tile)
            if existing is None:
                tileset.add_tile(tile)

    return tileset

Integration Points

flatten_layers() Check isinstance(layer, TilemapLayer) → call render_to_pixels() instead of layer.pixels directly tools.py When tilemap active + tiles mode: Pen/Eraser/Fill/Rect/Line operate on grid cells instead of pixels app.py Track tilemap edit mode state, toggle via Shift+`. Route mouse events to tile-mode or pixel-mode handlers. timeline.py Tilemap layer icon (grid icon) instead of normal layer icon. Right-click menu: "Convert to Pixel Layer" option. right_panel.py Mount TilesPanel when tilemap layer selected, hide when pixel layer selected. project.py v3 Serialize tilesets (tiles as PNG data), tilemap grids (2D array of uint32 packed refs). Backward compat: v3 reader, v2/v1 can't load tilemap layers. animation.py TilemapLayer synced across frames like regular layers. Each frame can have different tile grid but same tileset. Layer effects Work on tilemap layers too — effects applied AFTER render_to_pixels() during flatten.
