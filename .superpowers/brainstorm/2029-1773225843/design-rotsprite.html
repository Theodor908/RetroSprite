Feature 1: RotSprite Rotation — Detailed Design

Pixel-art-aware rotation with on-canvas handles

Algorithm: RotSprite

INPUT:  pixel_data (H×W×4 uint8 RGBA), angle (float degrees), pivot (x, y)

STEP 1 — Scale2x Upscale (×8)
  Apply modified Scale2x algorithm 3 times (2³ = 8× magnification)

  Scale2x for each pixel E with neighbors:
      A B C
      D E F        →  produces 2×2 block:  [E0, E1]
      G H I                                [E2, E3]

  Rules:
    E0 = (D==B && D!=H && B!=F) ? D : E
    E1 = (B==F && B!=D && F!=H) ? F : E
    E2 = (D==H && D!=B && H!=F) ? D : E
    E3 = (H==F && H!=D && F!=B) ? F : E

  RotSprite modification: use "similar" instead of "=="
  similar(a, b) = color_distance(a, b) < threshold
  This preserves anti-aliased edges and gradients

STEP 2 — Rotate at 8× resolution
  Standard nearest-neighbor rotation around pivot × 8
  At 8× resolution, nearest-neighbor produces clean results
  because Scale2x already created smooth sub-pixel edges

STEP 3 — Downscale back to 1×
  Area-average downsample 8×8 blocks → 1 pixel
  For pixel art: use mode (most common color) instead of average
  to avoid introducing new colors not in the palette

OUTPUT: rotated pixel_data, same dimensions, RGBA

Fast Rotation (Alternative)

Simple nearest-neighbor rotation at native resolution. Faster but produces jagged edges. User can toggle between RotSprite and Fast in the context bar.

For 90°/180°/270° rotations, both algorithms produce identical results (simple index transpose). RotSprite only matters for arbitrary angles.

Canvas Interaction

Activation

  • Select content (selection tool or Ctrl+A for whole layer)
  • Edit menu → Rotate, or keyboard shortcut (Ctrl+T)
  • Rotation handles appear around selection bounds

Handle Behavior

  • Corner handles: drag to rotate (cursor changes to ↻)
  • Shift+drag: snap to 15° increments (0, 15, 30, 45...)
  • Pivot point: visible dot at center, draggable
  • Live preview updates as you drag

Context Bar

  • Angle input field (type exact value, e.g. 33.5°)
  • Algorithm dropdown: RotSprite | Fast
  • Pivot X/Y coordinate fields

Confirm / Cancel

  • Enter → bake rotation into layer pixels (destructive)
  • Escape → cancel, restore original
  • Undo (Ctrl+Z) works after baking
  • Undo state saved before entering rotation mode

Performance Considerations

8× upscale on a 64×64 sprite = 512×512 intermediate. This is fine.
8× upscale on a 256×256 sprite = 2048×2048 intermediate. ~16MB. Still manageable but slower.

Mitigation: Only rotate the selection bounding box, not the full canvas. For large selections, show "Fast" preview during drag, compute RotSprite on mouse release. Cache the RotSprite result until angle changes.
