Non-Destructive Layer Effects — Full Set

Core + Advanced effects for RetroSprite Batch 4

Core Effects

1

Outline

Algorithm: For each transparent pixel adjacent to an opaque pixel, fill with outline color. Check 4-connected (cardinal) or 8-connected (diagonal too) neighbors.

Params: Color, thickness (1-4px), mode (outer/inner/both), 4-conn vs 8-conn

Pixel art note: Always 1px steps, no anti-aliasing. Inner outline erodes visible area.

2

Drop Shadow

Algorithm: Copy layer alpha mask, offset by (dx, dy), fill with shadow color at given opacity. Optionally apply small Gaussian blur (1-3px kernel). Composite behind the layer.

Params: Offset X/Y (-16 to +16), color, opacity (0-100%), blur radius (0-3)

Pixel art note: Blur radius 0 = hard pixel shadow (most common). Blur 1-3 = soft shadow.

3

Inner Shadow

Algorithm: Invert the alpha mask, offset by (dx, dy), optionally blur, then clip to original alpha. Result is shadow visible only inside opaque areas, near edges.

Params: Offset X/Y, color, opacity, blur radius

Pixel art note: Great for giving depth to flat sprites. Hard shadow (blur=0) works best at small sizes.

Color Effects

4

Hue / Saturation Shift

Algorithm: Convert each pixel RGB → HSV, add hue offset (0-360°), multiply saturation by factor, adjust value/lightness, convert back to RGB. Preserve alpha.

Params: Hue shift (-180° to +180°), saturation factor (0-200%), lightness offset (-100 to +100)

Pixel art note: Works on NumPy arrays via vectorized colorsys or manual RGB↔HSV formulas. No new colors introduced if palette is small.

5

Gradient Map

Algorithm: Convert each pixel to luminance (0.299R + 0.587G + 0.114B). Use luminance as index into a gradient (list of color stops). Map darkest → first stop, brightest → last stop, interpolate between.

Params: Gradient (2+ color stops with positions), blend opacity

Pixel art note: Perfect for palette-swapping effects. No interpolation artifacts since we map to discrete palette colors.

Advanced Effects

6

Glow / Bloom

Algorithm: Extract bright pixels above threshold. Apply Gaussian blur (σ = radius). Composite blurred brightness back using Screen or Addition blend mode.

Steps: 1) Threshold filter → bright mask. 2) Gaussian blur on mask. 3) Screen-blend onto original.

Params: Threshold (0-255), radius (1-8), intensity (0-200%), color tint

Pixel art note: At small sprite sizes, even radius 2-3 gives noticeable glow. Can tint the glow color for neon effects.

7

Pattern Overlay

Algorithm: Tile a small pattern image (e.g., 4×4 crosshatch, scanlines, noise) across the layer bounds. Blend with configurable mode (multiply, overlay, etc.) and opacity. Clip to layer alpha.

Params: Pattern (built-in or custom), blend mode, opacity, scale (1x-4x), offset X/Y

Pixel art note: Built-in patterns: scanlines, crosshatch, dots, diagonal lines, noise, checkerboard. Great for retro CRT / texture effects.

Rendering Pipeline

For each layer (bottom to top):
  1. Get layer pixels (raw)
  2. Apply layer effects in order:
     a. Color effects (hue/sat, gradient map)  — modify pixel colors
     b. Pattern overlay                        — blend pattern onto pixels
     c. Glow/bloom                             — add glow composite
     d. Outline                                — expand canvas, add outline pixels
     e. Drop shadow                            — expand canvas, add shadow behind
     f. Inner shadow                           — add shadow inside alpha
  3. Composite onto frame using blend mode + opacity

Effects are stored as data on the Layer object, applied during flatten_layers(), never modify source pixels.
